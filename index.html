Closest Vector Problem Encryption

What is happening?

This is a form of encryption that is able to protect against quantum computers and is able to be preformed on traditional computers. This is an example of using the cloest vector problem to encrypt data heres how it works at a high level.

  1. Each person has a good and bad set of vectors that make a lattice.
    The good and bad sets both make the same lattice, but the bad set makes traversing it way harder
  2. Your message is then converted into a number representation. I am using pairs of charcters converted to ASCII
  3. That representation has a bit of noise added to it.
  4. The noisy representation is then put onto the lattice of the intended recipient as a single point.
  5. The recipient then uses their good vectors to find the point on their lattice that is closest to that noisy point
  6. Reverse the string to integer operation to get the message. This step can involve error correction as messages are not always perfectly preserved

In this simple CVP based encryption scheme you use the good basis as the private key so only the recipient knows what it is but the sender can make and put a point on the lattice formed by the good basis becuase there is a bad basis that still makes the same lattice. This means that everyone knows the bad basis and can try and use it to solve the problem and in 2 dimensions it is not a huge challenge. Now think about this in 3 dimensions, it gets a bit more complex. Now lets multiply that by 100 and you get the small end of the range of dimensions that cryptographers are using.

Why is this safe against quantum computers?

Quantum computers are able to solve RSA encryption so fast with the use of Shor's algorithm. With encryption schemes that are based on CVP there is no algoithm that exists for classic computers or quantum computers that can solve it in polynomial time.
